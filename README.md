# PinFit — Pinterest 코디 쇼핑 어시스턴트

내 Pinterest 보드/핀에 저장된 코디, 또는 직접 업로드한 코디 사진을 AI가 분석해서
무신사 · 29CM · W컨셉 · 지그재그에서 비슷한 아이템을 찾아주고, 내가 설정한
**아이템 단가(개당 최대 가격)** 이하로 필터링해서 보여주는 앱입니다.

## 실행 방법

```bash
npm install
cp .env.example .env   # 아래 "환경변수 설정" 참고해서 값 채우기
npm run dev
```

`http://localhost:3002` 접속.

## 환경변수 설정

### OpenAI API 키 (이미지 분석 — 이거 하나만 있으면 됩니다)

1. https://platform.openai.com/api-keys 에서 API 키 발급
   (OpenAI는 결제 정보 등록 + 최소 충전이 필요한 종량제입니다. gpt-4o-mini는 이미지 1장당
   비용이 매우 저렴한 편이에요.)
2. `.env`의 `OPENAI_API_KEY`에 붙여넣기
3. 서버 재시작

**끝이에요.** Pinterest 개발자 앱은 필요 없습니다 (아래 참고).

### (선택) 매칭 정확도를 높이는 키 2개

기본 기능은 위 OpenAI 키만으로 동작하지만, 아래 키를 넣으면 "사진 속 그 옷과 똑같은
상품"을 찾는 정확도가 크게 올라갑니다. 둘 다 없으면 자동으로 기존 방식으로 폴백합니다.

- **네이버 쇼핑 API** (무료) — 후보 상품을 수백 개 몰에서 끌어와 매칭 품질을 높임.
  https://developers.naver.com 에서 앱 등록 → "검색" API 사용 설정 → `NAVER_CLIENT_ID`,
  `NAVER_CLIENT_SECRET`를 `.env`에 입력.
- **구글 렌즈 시각 검색 (SerpApi)** — 코디마다 "📸 사진과 시각적으로 똑같은 상품" 섹션을
  추가 (네이버 렌즈 스타일). https://serpapi.com/manage-api-key 에서 발급(무료 월 100회) →
  `SERPAPI_KEY`를 `.env`에 입력. 단, 이미지가 공개 URL이어야 하므로 Pinterest 핀에는
  적용되지만 로컬에서 직접 업로드한 사진에는 적용되지 않습니다.

## Pinterest 연동 방식 (개발자 앱 불필요)

앱을 켜면 로그인 대신 **본인 Pinterest 아이디만 입력**하면 됩니다.
그러면 `https://www.pinterest.com/{아이디}/feed.rss` (핀터레스트가 공개 제공하는
RSS 피드)를 통해 내가 최근 저장한 핀들을 그대로 가져와 분석합니다.

- ✅ 개발자 앱 생성 / OAuth / 트라이얼 승인 **전부 불필요**
- ⚠️ **단, 프로필과 보드가 '공개(Public)'로 설정되어 있어야 합니다.** 비밀(secret)
  보드나 비공개 프로필의 핀은 RSS에 노출되지 않아 가져올 수 없습니다.
- 가져오는 것은 "내가 저장한 핀"입니다. 핀터레스트 홈에 뜨는 알고리즘 추천 자체를 실시간
  스크레이핑하는 API는 존재하지 않지만, 알고리즘을 보고 마음에 들어 저장한 핀 = 내 취향이
  므로 목적상 이 방식이 가장 정확합니다.

> **왜 OAuth 방식을 안 쓰나요?** 원래 Pinterest 공식 API(OAuth)로 구현했었는데,
> 2026년 기준 Pinterest가 개발자 앱 "트라이얼 액세스" 승인을 대부분 거부/지연시켜서
> (승인 안 나면 App secret 키조차 안 보임) 개인이 쓰기가 사실상 불가능해졌습니다.
> 그래서 승인이 필요 없는 공개 RSS 방식으로 전환했습니다. (OAuth용 코드는
> `server/pinterestClient.ts`에 그대로 남겨뒀으니, 나중에 앱 승인을 받으면 되살릴 수 있어요.)

## 4개 쇼핑몰 연동 방식 (중요, 정직하게 설명)

쇼핑몰마다 검색 결과를 서버에서 가져올 수 있는지가 달라서, 몰마다 동작 방식이 다릅니다.

| 쇼핑몰 | 방식 | 이유 |
|---|---|---|
| **무신사** | ✅ 실시간 상품 카드 (이름/가격/이미지/링크) | 검색 페이지가 서버에서 렌더링되며 상품 데이터가 그대로 포함되어 있어 안정적으로 가져올 수 있음 |
| **29CM** | 🔗 실시간 검색 결과 딥링크 | 검색 결과가 브라우저에서 JS로 그려져서 서버에서는 상품 데이터를 못 가져옴. 검색 API는 Cloudflare 봇 차단이 걸려 있음 |
| **W컨셉** | 🔗 실시간 검색 결과 딥링크 | 검색 페이지에 상품 개수/필터만 서버에서 내려오고, 실제 상품 목록은 인증된 내부 API로 브라우저에서만 불러옴 |
| **지그재그** | 🔗 실시간 검색 결과 딥링크 | 완전한 클라이언트 렌더링 앱(SPA)이라 서버에서는 빈 화면만 받아옴. 공개된 API는 입점 판매자 전용 |

딥링크 카드는 가짜 상품을 지어내는 대신, AI가 뽑아낸 검색어로 그 쇼핑몰의 실제 검색
결과 페이지로 바로 연결합니다 — 클릭하면 항상 그 순간의 진짜 재고/가격을 보게 됩니다.
(예전 버전은 AI가 가격/상품명을 그럴싸하게 "지어내고" 링크는 쇼핑몰 메인으로만 연결하는
방식이었는데, 이번에 그 방식을 걷어내고 실제 데이터 또는 실제 검색 링크만 보여주도록
바꿨습니다.)

나중에 29CM/W컨셉도 무신사처럼 실시간 카드로 업그레이드하려면 브라우저 개발자
도구 Network 탭에서 검색 시 호출되는 내부 API를 직접 캡처해서
`server/malls/29cm.ts` / `server/malls/wconcept.ts`의 `search()`를
`server/malls/musinsa.ts`와 같은 패턴으로 구현하면 됩니다.

## 기능

- **Pinterest 연동**: 내 아이디를 입력하면 최근 저장한 핀을 GPT Vision(OpenAI)이
  분석해서 상의/하의/아우터/신발/가방/액세서리로 나누고, 각각 4개 몰에서 매칭 아이템을 찾음
- **사진 업로드**: Pinterest 없이도 아무 코디 사진이나 올리면 동일하게 분석
- **아이템 단가 설정**: 슬라이더로 "아이템 1개당 최대 가격"을 정하면 그 가격 이하 상품만
  기본으로 보이고, 초과 상품은 접혀서 필요할 때만 펼쳐볼 수 있음. 카테고리별로 다르게
  설정도 가능 (예: 아우터는 15만원, 상의는 5만원)
- 가격 슬라이더/쇼핑몰 선택은 전부 브라우저에서 즉시 반영되며, 재분석(OpenAI 재호출)이
  필요 없음

## 알려진 제약

- 첫 동기화 시 최대 12개 핀만 분석합니다 (OpenAI 호출 비용/속도 고려). 더 늘리고
  싶으면 `src/App.tsx`의 `MAX_PINS` 값을 조정하세요.
- 무신사 스크레이핑은 무신사가 검색 페이지 구조를 바꾸면 깨질 수 있습니다. 이 경우
  `server/malls/musinsa.ts`가 조용히 실패하고 자동으로 딥링크 카드로 대체되므로 앱 자체가
  죽지는 않습니다.
